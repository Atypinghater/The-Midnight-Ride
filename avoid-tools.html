Tools to Avoid

Vortex: Flawed conflict management which makes it annoying to use, does not track generated files and generally slow.


Nexus Mod Manager/Fallout Mod Manager: Outdated/abandoned, have no virtual file system or even dynamic movement of mods with loose files.


Mod Organizer 1: There is no reason to use Mod Organizer 1 over Mod Organizer 2.


LOOT: As LOOT is automated, it will never fully be able to account for all the nuances of a load order, especially a large one. Incorrect load order can lead to overwriting or breaking features from mods.


Wrye Bash/Flash and Mator Smash: Similarly to LOOT, these programs are fully automated and will never be fully accurate. They will solve things that they may see as conflicts, but it reality they are intentional overwrites. They will also often miss critical conflicts.


zEdit/zClean: No ESL support. zEdit saves plugins with an old plugin format and neither have any real advantages over xEdit, and lack some essential features/support than xEdit has.


Merge Plugins Standalone/zMerge: Automated merging can potentially cause major breakage in mods, especially heavily-scripted ones. If you are close to the 254 plugin cap you should look to start flagging them as ESL with the xEdit SetESL script, while the program is ran with -PseudoESL. Odds are that many mods are not fully compatible with one another.


Ordenador/DDSopt: Breaks many textures by needlessly converting/adding/resizing alpha channels, cubemaps, and mipmaps. Use Cathedral Assets Optimizer instead.


Fallout 4 Configuration Tool: Outdated, use BethINI if you need a GUI to edit your INIs.
